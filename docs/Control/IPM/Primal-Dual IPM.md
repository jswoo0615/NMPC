### 실시간 최적 제어 (RT-NMPC)를 위한 Primal - Dual IPM 명세서
Interior-Point Method (IPM)는 물리적 제약 조건 (조향각 한계, 장애물 회피 등)이 존재하는 비선형 동역학 최적화 문제 (NLP)를 해결하는 가장 강력한 엔진입니다. 하지만 자율주행 차량의 실시간 제어를 담당하는 우리의 아키텍처는 무한한 연산 시간을 가정하는 '고전적 IPM'이 아닌, 속도와 강건성을 극대화한 **현대적인 Infeasible Primal-Dual IPM**을 채택하고 있습니다.

#### 1. Barrier와 Penalty의 차이점 및 역할
IPM의 핵심은 제약 조건을 목적 함수 내부로 흡수시키는 것에 있습니다. 이때 패널티와 배리어를 엄격하게 구분해야 합니다. 
- **2차 패널치 (Quadratic Penalty, $c(x)^{2}$)** : 경계선을 넘어가면 위반량의 제곱만큼 벌점을 부여하는 '고무줄'입니다. 경계 침범을 일시적으로 허용합니다.
- **로그 배리어 (Logarithmic Barrier, $-\mu log(s)$)** : 변수가 경계선 (0)에 다가갈수록 값이 무한대 $(\infty)$로 치솟아 절대 선을 넘지 못하게 막는 '콘크리트 벽'입니다.
- **전략** : 부등식 제약 조건 (예 : $g(x) \le 0$)을 슬랙 변수를 도입해 $s > 0$으로 변환하고, 이 $s$에 로그 배리어를 씌워 장애물 및 물리적 한계라는 '절대적인 안전 영역 (Interior)'을 강제합니다.

#### 2. 수학적 심장 : KKT 조건과 중앙 경로 (Central Path)
우리의 솔버는 목적 함수를 변환해두고 무작정 수렴하기를 기다리는 고전적 방식을 사용하지 않습니다.
##### A. 변형된 KKT 시스템 (Modified KKT Conditions)
최적해는 기울기가 0이 되는 지점을 의미하는 KKT 조건을 만족해야 합니다. 우리는 이 중 상보적 여유성 (Complementarity) 조건을 배리어 파라미터 $\mu$를 포함하도록 변형해야 합니다.

$$s_{i} \cdot \lambda_{i} = \mu$$
(슬랙 변수 $s$와 쌍대 변수 $\lambda$의 곱이 $\mu$와 같아야 함)
##### B. 뉴턴법을 통한 직접 풀이 (Primal-Dual Approach)
우리는 이 변형된 KKT 방정식 전체를 가우스-뉴턴 (Gauss-Newton) 근사가 적용된 **Riccati 솔버 ($O(H)$ 시간 복잡도) 으로 직접 풉니다.**  상태 변수 ($x$, Primal)와 쌍대 변수 ($\lambda$, Dual)를 동시에 업데이트하며 해를 찾아갑니다.
##### C. 동적 $\mu$ 업데이트 (Central Path 추종)
$25ms$ 한계 내에서 해를 찾기 위해, 매 반복 (Iteration)마다 중앙 경로를 따라 $\mu$를 기계적으로 깎아냅니다. 위 코드는 현재의 평균 갭 (Average Gap)에 기반하여 단일 스텝마다 $\mu$를 $0.2$배로 공격적으로 감소시키며 최적해 $(\mu \rightarrow 0)$에 접근합니다.

#### 3. Infeasible 초기화 허용
이 부분이 1960년대 교과서와 현대 자율주행 제어기를 가르는 가장 대표적인 차이점입니다.
- **고전적 한계** : 초기 추정 해 $x^{(0)}$가 모든 제약 조건을 완벽히 만족하는 '안전한 내부 (Interior)'여야만 시작할 수 있습니다. 만약 차량이 이미 장애물 영역을 살짝 침범했거나, 차선을 밟은 상태에서 제어기가 켜지면 솔버는 바로 발산합니다.
- **Infeasible Primal-Dual Interior-Point Method** : 차량의 물리적 상태 $(x)$나 제어 입력 $(u)$이 일시적으로 제약 조건을 위반 (Infeasible)한 상태이더라도 솔버는 멈추지 않습니다. 오직 수학적 가상 변수인 내부 슬랙 변수 $(s)$와 쌍대 변수 $(\lambda)$만 강제로 양수 $( > 0)$로 초기화 (Interior)해 주면, 솔버가 역으로 물리적 궤적 $(x)$의 멱살을 잡고 제약 조건 내부로 끌고 들어옵니다.

#### 4. 궤적 최적화 (OCP)와의 코드 맵핑 체계
이 코드에서는 IPM 요소들은 다음과 같이 물리적 현실과 직결됩니다.

| **수학적/최적화 개념**       | **동역학 시스템 및 코드 구현**                   | **아키텍처 내 의미**                                                           |
| :------------------- | :------------------------------------ | :---------------------------------------------------------------------- |
| **Primal Variables** | 상태 벡터 $(x_{k})$, <br>제어 입력 $(u_{k})$  | 차량의 조향각, 가속도, 위치, 속도 등 <br>물리적 제어 대상                                    |
| **Dual Variables**   | 제약 조건 승수 $(\lambda)$, <br>슬랙 $(s)$    | 제약 조건을 어기려 할 때 발생하는 반발력의 <br>수학적 크기                                     |
| **Objectives**       | 행렬 $Q, \ R$ 기반의 단계 비용 $(L)$           | 추종 오차 최소화 <br>승차감 (조향 억제)을 위한 기본 타협점                                    |
| **Constraints**      | 동역학 $(x_{k+1} = Ax + Bu)$, <br>장애물 회피 | 선형 적분기 예측<br>장애물 여유 거리 (`Obstacle_Margin`)                              |
| **Merit Function**   | $L1$ 잔차 + $log$ 배리어 패널티               | Line Search의 브레이크, 물리적 예측성, <br>제약 준수, 비용 최소화를 단일 스칼라 값으로<br>검증하는 최종 판사 |
