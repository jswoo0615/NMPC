#### 0. 질문

차가 장애물에서 5m 떨어져 있어 매우 안전하더라도 ($c = -5$), 솔버 내부의 슬랙 변수가 $s = 3$으로 계산되었다면 $|-5 + 3| = |-2| = 2$가 되어 2000 ($w_{L1} \times 2$)이라는 엄청난 벌점을 부여합니다.

여기서 $s = 3$으로 계산되었다면에서 질문이 있습니다
슬랙변수라 함은 $g(x) + s \ge 0$과 같은 형태인데, s = 3이라 하면 $g(x) \ge -3$ 등과 같이 제약조건이 이루어진 상태에서 값이 계산되는 것 같은데, 풀이과정이 맞는지요?

---
#### 1. 수식의 교정 : 부등식이 아니라 등식입니다.

슬랙 변수를 도입하는 과정을 $g(x) + s \ge 0$ 형태의 부등식으로 나타낼 수 있고, 그래서 $s = 3$일 때 제약 조건이 $g(x) \ge -3$ 으로 변형된다고 추론했습니다. 직관적인 접근이었으나, IPM의 실제 수학적 변환은 다릅니다.

물리적 제약 조건 (예 : 안전 마진 침범)이 $c(x) \le 0$이라고 할 때, IPM은 이를 풀기 위해 가상의 양수 변수 $s (s > 0)$을 더해서 억지로 완벽한 등식 (Equality)으로 만들어 버립니다.

$$c(x) + s = 0 \quad \text{단, } s > 0$$
- **정상적인 (일관된) 상태** : 차가 장애물에서 5m 떨어져 있어 $c(x) = -5$ 라면, 수식이 $0$이 되기 위해 슬랙 변수 $s$는 반드시 정확히 $5$여야 합니다. $(-5 + 5 = 0)$
#### 2. 그렇다면 왜 $c = -5$인데 $s = 3$이 나오는가?

여기서 Primal-Dual IPM의 가장 핵심적인 특징이 등장합니다.
솔버의 뇌 속에서는 물리적 상태인 $x$ (차량의 위치 등)와 가상의 변수인 $s$ (슬랙 변수)가 **서로 완전히 독립적인 인격체**처럼 별개로 계산되고 업데이트됩니다.

1. **Newton Step (업데이트 방향 계산)** : 솔버는 현재 상태에서 $x$를 얼마나 바꿀지 $(\Delta x)$, 그리고 $s$ 얼마나 바꿀지 $(\Delta s)$를 연립방정식 ([[NVIDIA/Algorithms_for_Control/NMPC/docs/Solver/RiccatiSolver|RiccatiSolver]])을 통해 동시에 계산합니다.
2. **Line Search (보폭 $\alpha$ 적용)** : 그리고 이 두 변수는 동일한 보폭 $\alpha$를 가지고 다음 스텝으로 이동합니다.
	- 새로운 물리적 상태 : $x_{new} = x + \alpha \Delta x \rightarrow$ 이로 인해 $c(x_{new}) = -5$가 됨
	- 새로운 슬랙 변수 : $s_{new} = s + \alpha \Delta s \rightarrow$ 이 수식의 결과로 $s_{new} = 3$이 됨

왜 둘이 안 맞을까요? **$c(x)$는 비선형 (Non-linear) 함수**인데 반해, 솔버가 계산한 $\Delta x$와 $\Delta s$는 선형화 (Linearized Jacobian)된 근사치를 바탕으로 구한 것이기 때문입니다. 곡선을 직선으로 착각하고 크게 한 걸음 $(\alpha)$을 내디뎠더니, 물리적 현실 $(c = -5)$과 솔버의 수학적 예상 $(s = 3)$ 사이에 $2$만큼의 오차 (Inconsistency)가 발생해 버린 것입니다.

#### 3. L1 Penalty의 개입 : "거짓말하지 마라"
위의 예시인 $|-5 + 3| = |-2| = 2$의 진짜 의미가 명확해집니다.
차가 장애물에서 5m 떨어져 있으니 $(c = -5)$ 물리적으로는 사고가 나지 않았고 매우 안전합니다.
하지만 수학적으로는 $c(x) + s = 0$이라는 절대 규칙이  $-5 + 3 = -2 \neq 0$ 으로 깨져버렸습니다. 솔버의 물리 세계와 수학 세계가 동기화되지 않고 '거짓말'을 하고 있는 상태입니다.
- **[[evaluate_merit]] 함수 (L1 Penalty)** : "차가 안전한 것은 알겠으나, 하지만 네가 계산한 물리 상태 $(c)$와 가상 상태 $(s)$의 차이가 2만큼 맞지 않다. 수치해석이 붕괴하고 있다는 뜻이다. 패널티를 2000 부여! 이 보폭 $(\alpha)$은 폐기하고 다시 계산!"

#### 결론
사용자의 추론처럼 제약 조건의 경계선이 $g(x) \ge -3$으로 유연하게 변하는 것이 아닙니다.
오히려 반대로, "물리 법칙 $c(x)$와 수학적 가상 변수 $s$는 반드시 $c(x) + s = 0$으로 완벽하게 결합되어야만 하며, 이것이 조금이라도 어긋나는 순간 (예 : $c = -5$, $s = 3$) 엄격한 L1 패널티를 부과하여 선형화 오차로 인한 궤도 이탈을 막는다"는 것이 정확한 풀이 과정입니다.

슬랙 변수는 제약조건에 관련된 것이잖아요?

---
#### 추가 질문 답변
이론적으로는 100% 맞습니다. 하지만 종이에 적힌 수학을 'C++ 수치해석 솔버 (Numerical Solver)'로 옮기는 순간, 이 둘은 서로 모르는 남남으로 분할됩니다.
#### 1. 솔버의 구조 : 차원 확장 (Lifting)
만약 슬랙 변수가 제약 조건에 완전히 묶여 있다면, 솔버는 그냥 $s$ 자리에 $-c(x)$를 넣고 풀면 그만일 것입니다.
$$-\mu \ log(-c(x))$$
하지만 이렇게 하면 $c(x)$가 비선형 (Non-linear)일 경우, 로그 함수 안에 비선형 함수가 갇혀버려서 자코비안과 헤시안 행렬이 미친 듯이 꼬여버립니다. $25ms$ 안에 풀 수 없는 끔찍한 연산이 됩니다.

그래서 IPM (내부점 기법) 설계자들은 다른 방향으로 접근합니다.
**"슬랙 변수 $s$를 $x$에 종속된 값이 아니라, $x$와 동급인 '완전히 독립적인 최적화 변수'로 승격시키자!"**
- **원래 변수** : $x$ (차량의 상태, 조향, 가속도 등)
- **IPM 변수** : $Z = [x, s, \lambda]^{T}$

이제 솔버의 눈에 $x$와 $s$는 그냥 배열 인덱스 `Z[0]`과 `Z[1]`일 뿐입니다. 둘은 완전히 독립적으로 움직일 권리를 얻었습니다.
#### 2. 선형화의 배신 (Linearization Error)
독립된 두 변수 $x$와 $s$를 데리고, 솔버는 다음 스텝을 밟기 위해 뉴턴법 (Newton's Method)을 사용합니다. 뉴턴법은 복잡한 곡선 세계를 무식하게 '직선 (접선)'으로 퉁쳐서 예측하는 방식입니다.

솔버는 연립방정식을 풀어 두 변수의 독립적인 보폭 $(\Delta x, \Delta s)$을 계산합니다.
- $x_{new} = x + \alpha \Delta x$
- $s_{new} = s + \alpha \Delta s$

여기서 비극이 발생합니다.
- $s$는 원래 가상의 변수라 **직선**으로 완벽하게 예측대로 움직입니다.
- 하지만 $x$가 움직여서 만들어내는 물리적 현실 $c(x_{\text{new}})$는 **비선형 곡선**입니다.
직선으로 갈 줄 알고 스텝 $(\alpha)$를 크게 밟았는데, 실제 물리 세계 $(c(x))$는 휘어져 버렸으니 두 독립 변수 간의 약속이었던 $c(x_{new}) + s_{new} = 0$이 깨져버리는 것입니다.
#### 3. L1 Penalty의 진짜 역할 : 고무줄
정리하자면, **슬랙 변수 $s$는 태생적으로는 제약 조건 $c(x)$에서 튀어나온 것이 맞지만, 최적화를 빨리 풀기 위해 솔버 내부에서는 '독립적인 자유 변수'로 풀려난 상태**입니다.
자유를 얻은 두 변수 ($c(x)$와 $s$)가 선형화 오차 때문에 각자 다른 곳으로 찢어지려 할 때, 이 둘을 강제로 묶어주는 **무자비한 수학적 고무줄**이 바로 L1 Penalty 항입니다.
$$w_{L1} |c(x_{new}) + s_{new}|$$
